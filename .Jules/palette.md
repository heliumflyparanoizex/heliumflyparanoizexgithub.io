## 2024-05-24 - Focus Visibility using Utility Classes
**Learning:** Adding accessible focus states to interactive UI elements that lack them is critical for keyboard navigation. Using existing Tailwind utility classes (e.g., `focus-visible:ring-2`, `focus-visible:outline-none`, `focus-visible:ring-{color}`) provides an immediate, semantic way to introduce focus visibility without creating custom CSS overrides, keeping the styling tightly coupled to the component architecture.
**Action:** When working in codebases that utilize Tailwind CSS, always prioritize `focus-visible` utility classes for accessible focus styling on interactive elements (buttons, links) rather than relying on custom CSS styles.

## 2024-05-24 - Skip to Main Content Link
**Learning:** For single-page applications with heavy top navigation or visual elements (like videos and canvases), providing a "Skip to main content" link is a critical accessibility feature. It allows keyboard and screen reader users to bypass repetitive navigation and go straight to the primary content. Using Tailwind utility classes like `sr-only focus:not-sr-only focus:absolute z-[9999]` ensures it remains visually hidden until explicitly focused, maintaining the visual design while improving UX.
**Action:** Always ensure a "Skip to main content" link is present immediately after the `<body>` tag in applications with significant navigation headers. Use utility classes to handle the focus visibility gracefully.
