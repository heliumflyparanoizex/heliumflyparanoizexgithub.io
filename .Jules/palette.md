## 2024-05-24 - Focus Visibility using Utility Classes
**Learning:** Adding accessible focus states to interactive UI elements that lack them is critical for keyboard navigation. Using existing Tailwind utility classes (e.g., `focus-visible:ring-2`, `focus-visible:outline-none`, `focus-visible:ring-{color}`) provides an immediate, semantic way to introduce focus visibility without creating custom CSS overrides, keeping the styling tightly coupled to the component architecture.
**Action:** When working in codebases that utilize Tailwind CSS, always prioritize `focus-visible` utility classes for accessible focus styling on interactive elements (buttons, links) rather than relying on custom CSS styles.

## 2024-05-25 - Skip to Main Content Link Pattern
**Learning:** For single-page applications or sites with heavy top navigation, keyboard and screen reader users can be bogged down tabbing through all navigation items on every refresh or focus reset. Adding a 'Skip to main content' link that is visually hidden (`sr-only`) but visible on focus (`focus:not-sr-only focus:absolute z-[9999]`) is a standard, crucial accessibility feature. Doing this via Tailwind utility classes directly on the element (e.g. `bg-black text-white p-4 font-bold`) provides high visibility contrast without needing custom CSS.
**Action:** Always verify the presence of a "Skip to main content" link right after the opening `<body>` tag, pointing directly to the primary `<main>` or hero section ID (like `#inicio`). Use existing Tailwind visibility and absolute positioning classes for robust keyboard interaction.
