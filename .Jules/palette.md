## 2024-05-24 - Focus Visibility using Utility Classes
**Learning:** Adding accessible focus states to interactive UI elements that lack them is critical for keyboard navigation. Using existing Tailwind utility classes (e.g., `focus-visible:ring-2`, `focus-visible:outline-none`, `focus-visible:ring-{color}`) provides an immediate, semantic way to introduce focus visibility without creating custom CSS overrides, keeping the styling tightly coupled to the component architecture.
**Action:** When working in codebases that utilize Tailwind CSS, always prioritize `focus-visible` utility classes for accessible focus styling on interactive elements (buttons, links) rather than relying on custom CSS styles.
## 2026-06-01 - [Skip to Main Content Link]
**Learning:** Single-page applications with heavy top navigation (like the extensive sticky menu in this app) severely degrade the keyboard and screen-reader experience if users are forced to tab through every nav link on every page load/interaction.
**Action:** Always include a "Skip to main content" link as the absolute first focusable element in the DOM (immediately after `<body>`). Use Tailwind's utility classes (e.g., `sr-only focus:not-sr-only focus:absolute z-[9999] bg-black text-white p-4 font-bold`) to ensure it remains visually hidden for mouse users but clearly appears and is usable for keyboard navigators.
## 2026-06-01 - Adding 'Skip to main content' link
**Learning:** For SPAs with heavy top navigation, implementing an accessible skip link improves keyboard navigation significantly. While standard Tailwind utility classes (like `sr-only focus:not-sr-only`) exist, in environments or setups where they dynamically fail on focus, a robust fallback using standard inline or local CSS in a global stylesheet is necessary for reliable accessibility.
**Action:** Always verify that 'skip to main content' links correctly receive visual focus and become visible upon Tab navigation, regardless of the CSS methodology used.
